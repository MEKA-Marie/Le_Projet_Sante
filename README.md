# Projet Santé Publique - Semaine 2 et 3

Ce dépôt contient une série de scripts Python réalisés dans le cadre d’un mini-projet de santé publique et d’exercices de programmation.

## Objectif du projet

Le but est de simuler des analyses utiles pour la gestion d’un système de santé :
- suivi de patients,
- estimation de la capacité hospitalière,
- analyse de la densité médicale,
- contrôle des stocks de médicaments,
- triage des urgences,
- comparaison de plusieurs établissements de santé.

## Fichiers présents

- sante_variable.py : fichier principal du mini-projet. Il contient les données d’hôpitaux, de médicaments, les calculs de densité médicale, de taux d’occupation et la classification des stocks.
- semaine2_exercice1_sante.py : fiche patient avec coût de consultation, taux d’occupation des lits et ratio consultations/médecin.
- semaine2_exercice2_sante.py : rapport trimestriel d’un hôpital avec indicateurs OMS, budget et projection.
- semaine2_challenge.py : challenge de la semaine 2 (à compléter ou à utiliser selon l’évolution du projet).
- semaine3_challenge.py : rapport comparatif entre plusieurs hôpitaux avec indicateurs clés et vérification du budget.
- semaine3_exercice1_stocks.py : rapport de stocks pharmaceutiques avec classification par niveau d’alerte.
- semaine3_exercice2_triage.py : système interactif de triage des urgences basé sur la température, la saturation, la tension et la douleur.
- semaine3_mini_projet.py : fichier de mini-projet de la semaine 3 (présentement vide, à compléter).

## Prérequis

- Python 3.x installé
- Un terminal ou un environnement Python compatible

## Exécution des scripts

Exécuter un fichier avec la commande :

```bash
python nom_du_fichier.py
```

Exemples :

```bash
python sante_variable.py
python semaine3_exercice2_triage.py
```

## Résumé des analyses réalisées

Le projet couvre plusieurs dimensions essentielles de la santé publique :
1. Analyse hospitalière
2. Analyse de stock des médicaments
3. Triage des urgences
4. Comparaison de structures de santé
5. Suivi de coûts et de ressources

## Notes

Ce projet est à visée pédagogique et sert de support pour pratiquer Python dans un contexte de santé.
